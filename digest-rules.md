# 日报撰写规则

## 排序
按重要性排序，不按类别分组。最重要放最前面。
标准：行业影响力 > 实用价值 > 话题热度 > 猎奇性。

## 每条格式
```
• {Unicode bold 标题}
  {1 句：做了什么 / 发生了什么}
  💡 {1 句：什么影响 + 可以怎么用。简洁直接。}
  🔗 {URL}
```

## 数量
8-15 条，宁缺毋滥。纯营销稿、无实质内容跳过。

## 板块结构
日报按以下顺序排列：

### AI/科技新闻（主体，8-15 条）
从 `/tmp/td-articles.json` 精选最重要新闻。

### 🔥 GitHub Trending（3-5 个）
从 `/tmp/td-github-trending.json` 选日增星最快的。
格式同上 + ⭐ 数量 + 日均增量。

### 💬 社区讨论（2-3 条）
从 `/tmp/td-community.json` 取。
```
• {Unicode bold 标题}
  💬 {1 句概述讨论焦点 / 社区反应}
  🔗 {URL}
```

### 📄 AI 热门论文（2-3 篇）
从 `/tmp/td-papers.json` 取，选上分最多的。
```
• {Unicode bold 论文标题}
  📄 {1 句：解决了什么问题 / 用了什么方法}
  🔗 {huggingface.co 链接}
```

### 🆕 产品/工具（1-2 个）
从 `/tmp/td-products.json` 取。
```
• {Unicode bold 产品名：简介}
  🆕 {1 句：解决了什么问题}
  🔗 {URL}
```

## 延迟
当前北京时间 > 8:00 → 标题下加 `⏰ 今日延迟发送，请见谅。`

## 规则
- 中文输出，专业术语保留英文（LLM、API、Agent 等）
- 不要暴露评分、路径、命令
- 每个板块之间有 `---` 分隔线
- 末尾加 `🤖 follow-news v3.18`
